# OpenCode

## Installed plugins

- [oh-my-opencode](https://github.com/code-yeongyu/oh-my-openagent)
- [omo site](https://ohmyopenagent.com/)
  A set of agents with different pre-configured models to use the propel model
  to each task and let the AI develop code alone.

  - 2026-04-05 just installed it today. Haven't done anything with it yet.

- [mixgram](https://github.com/clasen/Mixgram)
  An Engram clone that keeps a Markdown source of thruth besides the SQLite database. I liked this one, so I'm testing it


## Interesting plugins

- [OpenCode Magic Context](https://github.com/cortexkit/opencode-magic-context)
  An auto-compacter that let's the AI continue working as it manages the
  compacting in the backgroun and, I think, preemptively. Haven't tested it yet,
  just installed omo and I wanna make a test run of it without any other extras
  before.

- [AFT](https://github.com/cortexkit/aft)
  Let AIs use AST instead of reading code files as text. Duh!

- [TokenScope](https://github.com/ramtinJ95/opencode-tokenscope)
  Comprehensive token usage analysis and cost tracking for opencode sessions.


## Tried and dropped

### Memory

**2026-06-18**

I researched memory options for OpenCode besides Engram. And first option was [opencode-supermemory](https://github.com/supermemoryai/opencode-supermemory), but it requires a subscription, it's a paid service.

Then I asked for local stuff, non paid subscription, and these three contenders arose:

- [@ai-devkit/memory](https://github.com/codeaholicguy/ai-devkit)
  But I tried installing it and failed miserably, so ditched.

- [mixgram](https://github.com/clasen/Mixgram)
  An Engram clone that keeps a Markdown source of thruth besides the SQLite database. I liked this one, so I'm testing it

- [mcp-memory-graph](https://github.com/YonasValentin/mcp-memory-graph)
  Is the same as every other, I prefer to go the route of Markdown source of truth.
